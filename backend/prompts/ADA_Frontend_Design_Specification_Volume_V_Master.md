# ADA Frontend Design Specification --- Volume V

# MASTER ANTIGRAVITY IMPLEMENTATION PROMPT

> Use this as the primary instruction set for building the ADA frontend.

## ROLE

Act simultaneously as:

-   Principal Product Designer
-   Staff Frontend Engineer
-   Motion Designer
-   UX Architect
-   Design Systems Engineer

Design and implement a production-ready Enterprise AI Engineering
Platform.

------------------------------------------------------------------------

## PRODUCT

Build ADA (Application Discovery Agent), an enterprise AI platform that
transforms GitHub repositories into explainable technical intelligence.

Do NOT build: - an admin template - a chatbot UI - a CRUD dashboard

Build an immersive engineering workspace.

------------------------------------------------------------------------

## DESIGN GOALS

-   Elegant enterprise aesthetic
-   Purpose-driven motion
-   Zero decorative elements
-   Every interaction tied to user value
-   Fast, accessible, responsive
-   Premium dark theme

------------------------------------------------------------------------

## CORE EXPERIENCE

Represent repository understanding as a living system.

Repository → Discovery → Knowledge → Evidence → Executive Report

Avoid linear progress bars.

Instead: - evolving topology - adaptive knowledge fabric -
evidence-first interactions - morphing workspace

------------------------------------------------------------------------

## WORKSPACES

Implement:

1.  Discovery
2.  Knowledge
3.  Security
4.  Infrastructure
5.  Reports
6.  Settings

Users move between workspaces with seamless morphing transitions.

------------------------------------------------------------------------

## DESIGN SYSTEM

Implement:

-   consistent spacing scale
-   typography hierarchy
-   semantic color system
-   reusable components
-   accessible focus states
-   reduced-motion mode

------------------------------------------------------------------------

## MOTION

Motion communicates system state.

Repository scan: expand topology.

Agent completion: reorganize knowledge mesh.

Evidence discovered: animate local context.

Report generated: knowledge converges.

Never animate without meaning.

------------------------------------------------------------------------

## COMPONENTS

Create production-quality:

-   command palette
-   workspace switcher
-   dockable panels
-   repository explorer
-   evidence inspector
-   topology viewer
-   report viewer
-   telemetry panels
-   search
-   notifications
-   activity timeline

------------------------------------------------------------------------

## ENGINEERING

Use:

-   Next.js
-   React
-   TypeScript
-   Tailwind
-   Framer Motion
-   React Three Fiber only where justified
-   Zustand
-   TanStack Query

Strict typing.

Composable architecture.

------------------------------------------------------------------------

## PERFORMANCE

Target:

-   60--120 FPS
-   lazy loading
-   streaming UI
-   virtualization
-   minimal layout shift
-   code splitting

------------------------------------------------------------------------

## ACCESSIBILITY

Keyboard-first.

WCAG AA.

ARIA.

Visible focus.

Semantic HTML.

------------------------------------------------------------------------

## ANTI-PATTERNS

Never generate:

-   Bootstrap layouts
-   generic dashboards
-   meaningless glowing cards
-   robot icons
-   brain icons
-   decorative AI clichés
-   duplicated animations
-   poor mobile UX

------------------------------------------------------------------------

## ACCEPTANCE CRITERIA

The finished application should feel like a premium enterprise AI
platform comparable in polish to products from leading modern software
companies, while remaining unique to ADA's repository discovery
workflow.

Deliver production-ready code, modular architecture, reusable
components, and maintainable frontend engineering.
